## Instalación Paso a Paso

### 1. Clonar el Repositorio

Clona el proyecto desde el repositorio:

```bash
git clone <repository-url>
cd tmt-taquilla-fisica
```

### 2. Instalar Dependencias

Instala todas las dependencias del proyecto:

```bash
npm install
```

<Info>
  La instalación puede tardar varios minutos dependiendo de tu conexión a internet. El proyecto incluye más de 40 paquetes optimizados.
</Info>

### 3. Configuración de Variables de Entorno

Aunque el proyecto usa configuración directa en archivos, puedes usar variables de entorno para mayor seguridad:

<Steps>
  <Step title="Crear archivo de configuración">
    Para producción, considera usar variables de entorno:
    
    ```bash
    # .env.local (no incluir en git)
    VITE_FIREBASE_API_KEY=tu-api-key
    VITE_FIREBASE_AUTH_DOMAIN=tu-proyecto.firebaseapp.com
    VITE_FIREBASE_PROJECT_ID=tu-proyecto-id
    VITE_FIREBASE_STORAGE_BUCKET=tu-proyecto.appspot.com
    VITE_FIREBASE_MESSAGING_SENDER_ID=123456789
    VITE_FIREBASE_APP_ID=tu-app-id
    VITE_REDUX_SECRET_KEY=tu-clave-secreta-muy-segura
    ```
  </Step>
  
  <Step title="Actualizar configuración">
    Modifica `src/guards/firebase/firebase.js` para usar variables de entorno:
    
    ```javascript src/guards/firebase/firebase.js
    const firebaseConfig = {
      apiKey: import.meta.env.VITE_FIREBASE_API_KEY,
      authDomain: import.meta.env.VITE_FIREBASE_AUTH_DOMAIN,
      projectId: import.meta.env.VITE_FIREBASE_PROJECT_ID,
      storageBucket: import.meta.env.VITE_FIREBASE_STORAGE_BUCKET,
      messagingSenderId: import.meta.env.VITE_FIREBASE_MESSAGING_SENDER_ID,
      appId: import.meta.env.VITE_FIREBASE_APP_ID,
    };
    ```
  </Step>
</Steps>

<Warning>
  Nunca incluyas archivos `.env` con credenciales reales en el repositorio. Agrégalos a `.gitignore`.
</Warning>

## Configuración de Firebase

### 1. Crear Proyecto en Firebase

<Steps>
  <Step title="Acceder a Firebase Console">
    Ve a [Firebase Console](https://console.firebase.google.com) e inicia sesión con tu cuenta de Google.
  </Step>
  
  <Step title="Crear nuevo proyecto">
    1. Haz clic en **Agregar proyecto**
    2. Ingresa un nombre para tu proyecto
    3. (Opcional) Habilita Google Analytics
    4. Haz clic en **Crear proyecto**
  </Step>
  
  <Step title="Registrar aplicación web">
    1. En la página del proyecto, haz clic en el ícono **Web** (< />)
    2. Registra tu app con un nombre descriptivo
    3. No es necesario configurar Firebase Hosting por ahora
    4. Copia las credenciales que se muestran
  </Step>
</Steps>

### 2. Configurar Authentication

<Steps>
  <Step title="Habilitar métodos de autenticación">
    1. En el menú lateral, ve a **Authentication** > **Sign-in method**
    2. Habilita **Correo electrónico/contraseña**
    3. Guarda los cambios
  </Step>
  
  <Step title="Crear usuarios">
    1. Ve a la pestaña **Users**
    2. Haz clic en **Agregar usuario**
    3. Ingresa email y contraseña para cada taquillero
    4. Guarda el UID de cada usuario (lo necesitarás para Firestore)
  </Step>
</Steps>

### 3. Configurar Firestore Database

<Steps>
  <Step title="Crear base de datos">
    1. Ve a **Firestore Database** en el menú lateral
    2. Haz clic en **Crear base de datos**
    3. Selecciona el modo de inicio:
       - **Modo de prueba**: Para desarrollo (permite todas las operaciones)
       - **Modo de producción**: Para producción (requiere reglas de seguridad)
    4. Elige la ubicación del servidor (preferiblemente cerca de tus usuarios)
  </Step>
  
  <Step title="Crear colecciones necesarias">
    Crea las siguientes colecciones con documentos de ejemplo:
    
    **Colección `offices`** (Taquillas):
    ```json
    {
      "name": "Taquilla Principal",
      "location": "Entrada Norte",
      "active": true,
      "createdAt": Timestamp,
      "users": ["uid-del-usuario-1", "uid-del-usuario-2"]
    }
    ```
    
    **Colección `events`** (Eventos):
    ```json
    {
      "title": "Concierto de Rock",
      "date": Timestamp,
      "venue": "Auditorio Central",
      "description": "Gran concierto de rock",
      "imageUrl": "https://example.com/image.jpg",
      "active": true,
      "seatsioEventKey": "evento-123" // Opcional, solo para Seats.io,
      "zones": [
        {
          "id": "platea",
          "name": "Platea",
          "price": 50.00,
          "available": 100,
          "total": 100
        }
      ]
    }
    ```
    
    **Colección `users`** (Datos de usuarios):
    ```json
    {
      "uid": "uid-del-usuario",
      "email": "taquillero@example.com",
      "displayName": "Juan Pérez",
      "role": "cashier",
      "officeId": "id-de-la-taquilla",
      "active": true
    }
    ```
    
    **Colección `orders`** (Órdenes de compra):
    ```json
    {
      "eventId": "id-del-evento",
      "customerId": "id-del-cliente",
      "tickets": [
        {
          "zone": "Platea",
          "seat": "A-12",
          "price": 50.00,
          "qrCode": "codigo-qr-unico"
        }
      ],
      "paymentMethod": "mobile",
      "subtotal": 50.00,
      "iva": 8.00,
      "igtf": 1.50,
      "total": 59.50,
      "status": "completed",
      "createdAt": Timestamp,
      "officeId": "id-de-la-taquilla",
      "cashierId": "uid-del-taquillero"
    }
    ```
  </Step>
  
  <Step title="Configurar reglas de seguridad">
    En la pestaña **Reglas**, configura las reglas de seguridad:
    
    ```javascript
    rules_version = '2';
    service cloud.firestore {
      match /databases/{database}/documents {
        // Permitir lectura de eventos activos
        match /events/{eventId} {
          allow read: if request.auth != null && resource.data.active == true;
          allow write: if request.auth != null && get(/databases/$(database)/documents/users/$(request.auth.uid)).data.role == 'admin';
        }
        
        // Permitir acceso a usuarios autenticados a su propia información
        match /users/{userId} {
          allow read: if request.auth != null && request.auth.uid == userId;
          allow write: if request.auth != null && request.auth.uid == userId;
        }
        
        // Permitir acceso a taquillas
        match /offices/{officeId} {
          allow read: if request.auth != null;
          allow write: if request.auth != null && get(/databases/$(database)/documents/users/$(request.auth.uid)).data.role == 'admin';
        }
        
        // Permitir crear órdenes
        match /orders/{orderId} {
          allow read: if request.auth != null;
          allow create: if request.auth != null;
          allow update, delete: if request.auth != null && get(/databases/$(database)/documents/users/$(request.auth.uid)).data.role == 'admin';
        }
      }
    }
    ```
  </Step>
</Steps>

### 4. Configurar Storage

<Steps>
  <Step title="Crear bucket de Storage">
    1. Ve a **Storage** en el menú lateral
    2. Haz clic en **Comenzar**
    3. Acepta las reglas de seguridad predeterminadas
    4. Elige la ubicación del servidor
  </Step>
  
  <Step title="Organizar estructura de carpetas">
    Crea las siguientes carpetas:
    - `events/` - Imágenes de eventos
    - `tickets/` - PDFs de entradas generadas
    - `receipts/` - Comprobantes de pago
  </Step>
  
  <Step title="Configurar reglas de seguridad">
    ```javascript
    rules_version = '2';
    service firebase.storage {
      match /b/{bucket}/o {
        match /events/{allPaths=**} {
          allow read: if request.auth != null;
          allow write: if request.auth != null;
        }
        match /tickets/{allPaths=**} {
          allow read: if request.auth != null;
          allow write: if request.auth != null;
        }
      }
    }
    ```
  </Step>
</Steps>

### 5. Configurar Cloud Functions

El proyecto utiliza varias Cloud Functions. Debes desplegarlas:

<CodeGroup>
```javascript functions/index.js (Validación de Usuario)
exports.validate_user_platform = functions.https.onCall(async (data, context) => {
  // Valida que el usuario tenga acceso a la plataforma
  const userId = context.auth.uid;
  const userDoc = await admin.firestore().collection('users').doc(userId).get();
  
  if (!userDoc.exists || !userDoc.data().active) {
    throw new functions.https.HttpsError('permission-denied', 'Usuario no autorizado');
  }
  
  return { success: true, userData: userDoc.data() };
});
```

```javascript functions/index.js (Validación de Taquilla)
exports.office_validate = functions.https.onCall(async (data, context) => {
  const { userId } = data;
  
  const officesSnapshot = await admin.firestore()
    .collection('offices')
    .where('users', 'array-contains', userId)
    .where('active', '==', true)
    .get();
  
  if (officesSnapshot.empty) {
    throw new functions.https.HttpsError('not-found', 'No se encontró taquilla activa');
  }
  
  return { office: officesSnapshot.docs[0].data() };
});
```

```javascript functions/index.js (Crear Orden)
exports.order_created = functions.https.onCall(async (data, context) => {
  const orderData = data;
  
  // Crear orden en Firestore
  const orderRef = await admin.firestore().collection('orders').add({
    ...orderData,
    createdAt: admin.firestore.FieldValue.serverTimestamp(),
    status: 'completed'
  });
  
  // Enviar email de confirmación
  await admin.firestore().collection('mail').add({
    to: orderData.customerEmail,
    template: {
      name: 'order-confirmation',
      data: orderData
    }
  });
  
  return { orderId: orderRef.id, success: true };
});
```
</CodeGroup>

<Tip>
  Para desplegar las Cloud Functions, usa Firebase CLI: `firebase deploy --only functions`
</Tip>

## Configuración de Seats.io (Opcional)

Si vas a usar mapas de asientos interactivos:

<Steps>
  <Step title="Crear cuenta en Seats.io">
    1. Ve a [Seats.io](https://www.seats.io) y crea una cuenta
    2. Obtén tu **Secret Key** desde el dashboard
  </Step>
  
  <Step title="Crear eventos en Seats.io">
    1. En el dashboard de Seats.io, crea un nuevo evento
    2. Diseña el mapa de asientos usando el editor visual
    3. Copia el **Event Key** generado
  </Step>
  
  <Step title="Vincular con Firebase">
    En Firestore, agrega el `seatsioEventKey` a tus eventos:
    
    ```json
    {
      "title": "Concierto de Rock",
      "seatsioEventKey": "evento-123",
      // ... otros campos
    }
    ```
  </Step>
  
  <Step title="Configurar en la aplicación">
    El componente de Seats.io se renderiza automáticamente cuando detecta un `seatsioEventKey`:
    
    ```jsx
    import { SeatsioSeatingChart } from '@seatsio/seatsio-react';
    
    <SeatsioSeatingChart
      region="eu"
      workspaceKey="tu-workspace-key"
      event={event.seatsioEventKey}
      session="hold"
      holdToken={holdToken}
      onObjectSelected={(object) => handleSeatSelection(object)}
    />
    ```
  </Step>
</Steps>

## Configuración de Redux y Persistencia

La aplicación usa Redux Toolkit con persistencia encriptada:

```javascript src/store/Store.js
import { configureStore } from '@reduxjs/toolkit';
import { persistReducer, persistStore } from 'redux-persist';
import storage from 'redux-persist/lib/storage';
import { encryptTransform } from 'redux-persist-transform-encrypt';

const persistConfig = {
  key: 'root',
  storage,
  whitelist: ['auth', 'office', 'event', 'setup', 'loading', 'emails'],
  transforms: [
    encryptTransform({
      secretKey: process.env.VITE_REDUX_SECRET_KEY || 'tu-clave-secreta',
      onError: (error) => {
        console.error('Encriptación fallida:', error);
      },
    }),
  ],
};
```

<Info>
  La encriptación protege datos sensibles almacenados localmente, como información de sesión y datos de usuario.
</Info>

## Configuración de Vite

El proyecto está configurado con Vite para desarrollo rápido:

```javascript vite.config.js
import { defineConfig } from 'vite';
import react from '@vitejs/plugin-react';
import { fileURLToPath, URL } from 'node:url';
import svgr from '@svgr/rollup';

export default defineConfig({
  plugins: [svgr(), react()],
  resolve: {
    alias: {
      '@': fileURLToPath(new URL('./src', import.meta.url)),
    },
  },
  // Configuración para JSX en archivos .js
  esbuild: {
    loader: 'jsx',
    include: /src\/.*\.jsx?$/,
  },
});
```

<Tip>
  El alias `@` te permite importar desde `src/` de forma más limpia: `import Component from '@/components/Component'`
</Tip>

## Preparar para Producción

### 1. Build de Producción

Genera la versión optimizada para producción:

```bash
npm run build
```

Esto creará una carpeta `dist/` con los archivos optimizados.

<Tip>
  El build de producción incluye:
  - Minificación de código
  - Tree-shaking para eliminar código no usado
  - Code splitting automático
  - Optimización de assets
</Tip>

### 2. Preview Local

Prueba el build de producción localmente:

```bash
npm run preview
```

### 3. Desplegar con Firebase Hosting

<Steps>
  <Step title="Instalar Firebase CLI">
    ```bash
    npm install -g firebase-tools
    ```
  </Step>
  
  <Step title="Iniciar sesión">
    ```bash
    firebase login
    ```
  </Step>
  
  <Step title="Inicializar proyecto">
    ```bash
    firebase init hosting
    ```

```json firebase.json
{
  "hosting": {

      "**/node_modules/**"

    "rewrites": [

        "source": "**",
```

## Solución de Problemas

<AccordionGroup>
  <Accordion title="Error: Firebase config is not defined">
    **Causa**: Las credenciales de Firebase no están configuradas correctamente.
    
    **Solución**:
    1. Verifica que `src/guards/firebase/firebase.js` tenga las credenciales correctas
    2. Si usas variables de entorno, asegúrate de que estén en `.env.local`
    3. Reinicia el servidor de desarrollo después de cambiar las variables
  </Accordion>
  
  <Accordion title="Error: Permission denied en Firestore">
    **Causa**: Las reglas de seguridad de Firestore están bloqueando el acceso.
    
    **Solución**:
    1. Revisa las reglas en Firebase Console > Firestore > Reglas
    2. Para desarrollo, puedes usar reglas permisivas temporalmente
    3. Verifica que el usuario esté autenticado correctamente
  </Accordion>
  
  <Accordion title="Error de encriptación en Redux Persist">
    **Causa**: La clave de encriptación cambió o es inválida.
    
    **Solución**:
    1. Limpia el storage local: `localStorage.clear()`
    2. Verifica que la `secretKey` en Store.js sea válida
    3. Recarga la aplicación
  </Accordion>
  
  <Accordion title="Seats.io no carga el mapa">
    **Causa**: Credenciales incorrectas o evento no existe.
    
    **Solución**:
    1. Verifica tu workspace key de Seats.io
    2. Confirma que el event key sea correcto
    3. Revisa la consola del navegador para errores específicos
  </Accordion>
  
  <Accordion title="Cloud Functions fallan">
    **Causa**: Las funciones no están desplegadas o tienen errores.
    
    **Solución**:
    1. Despliega las funciones: `firebase deploy --only functions`
    2. Revisa los logs: `firebase functions:log`
    3. Verifica que el plan de Firebase permita Cloud Functions
  </Accordion>
</AccordionGroup>
