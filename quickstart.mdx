## Bienvenido a TMT - Taquilla Física

Esta guía te llevará desde la instalación hasta realizar tu primera venta de entrada, paso a paso.

<Info>
  **Tiempo estimado**: 15-20 minutos
</Info>

## Antes de Comenzar

Asegúrate de tener instalado:

- **Node.js** (versión 16 o superior)
- **npm** (incluido con Node.js)
- Una cuenta de **Firebase** con un proyecto configurado
- (Opcional) Una cuenta de **Seats.io** para eventos con asientos numerados

## Paso 1: Instalación y Configuración

<Steps>
  <Step title="Clonar el Repositorio">
    Clona el proyecto en tu máquina local:
    
    ```bash
    git clone <repository-url>
    cd tmt-taquilla-fisica
    ```
  </Step>
  
  <Step title="Instalar Dependencias">
    Instala todas las dependencias necesarias:
    
    ```bash
    npm install
    ```
    
    <Note>
      El proyecto incluye más de 40 paquetes optimizados para un rendimiento óptimo, incluyendo React 18, Firebase 10, Redux Toolkit, y Seats.io.
    </Note>
  </Step>
  
  <Step title="Configurar Firebase">
    Edita el archivo de configuración de Firebase en `src/guards/firebase/firebase.js`:
    
    ```javascript src/guards/firebase/firebase.js
    const firebaseConfig = {
      apiKey: "tu-api-key",
      authDomain: "tu-proyecto.firebaseapp.com",
      projectId: "tu-proyecto-id",
      storageBucket: "tu-proyecto.appspot.com",
      messagingSenderId: "123456789",
      appId: "tu-app-id",
    };
    ```
    
    <Warning>
      Nunca compartas estas credenciales públicamente. Considera usar variables de entorno para producción.
    </Warning>
  </Step>
  
  <Step title="Configurar Clave de Encriptación">
    En `src/store/Store.js`, actualiza la clave secreta para la encriptación de datos:
    
    ```javascript src/store/Store.js
    transforms: [
      encryptTransform({
        secretKey: 'tu-clave-secreta-muy-segura',
        onError: (error) => {
          console.error('Encriptación fallida:', error);
        },
      }),
    ],
    ```
    
    <Tip>
      Usa una clave fuerte y única. Esta clave se utiliza para proteger datos sensibles almacenados en Redux Persist.
    </Tip>
  </Step>
</Steps>

## Paso 2: Iniciar el Servidor de Desarrollo

Inicia la aplicación en modo desarrollo:

```bash
npm run dev
```

La aplicación se abrirá automáticamente en `http://localhost:5173`

<Info>
  TMT usa **Vite** como build tool, lo que garantiza un inicio ultrarrápido y hot-reload instantáneo.
</Info>

## Paso 3: Primer Inicio de Sesión

<Steps>
  <Step title="Acceder a la Aplicación">
    Abre tu navegador y ve a `http://localhost:5173`
    
    Verás la pantalla de inicio de sesión para taquilleros.
  </Step>
  
  <Step title="Configurar Usuario en Firebase">
    Antes de iniciar sesión, necesitas crear un usuario en Firebase:
    
    1. Ve a la consola de Firebase
    2. Navega a **Authentication** > **Users**
    3. Haz clic en **Add user**
    4. Ingresa email y contraseña
    5. En **Firestore Database**, crea una colección `users` con un documento para este usuario
  </Step>
  
  <Step title="Iniciar Sesión">
    Usa las credenciales del usuario que acabas de crear para iniciar sesión en la aplicación.
  </Step>
</Steps>

## Paso 4: Configurar Tu Primera Taquilla

<Steps>
  <Step title="Crear Documento de Taquilla">
    En Firestore, crea una colección `offices` con la siguiente estructura:
    
    ```json
    {
      "name": "Taquilla Principal",
      "location": "Entrada Norte",
      "active": true,
      "createdAt": "timestamp",
      "users": ["user-id-del-taquillero"]
    }
    ```
  </Step>
  
  <Step title="Validar Acceso">
    La aplicación utiliza la función `office_validate` para verificar que el usuario tenga acceso a una taquilla:
    
    ```javascript
    import { office_validate } from '@/guards/firebase/firebase';
    
    // Validación automática al iniciar sesión
    const result = await office_validate({ userId: currentUser.uid });
    ```
  </Step>
</Steps>

## Paso 5: Crear Tu Primer Evento

<Steps>
  <Step title="Agregar Evento en Firestore">
    Crea una colección `events` con un documento de ejemplo:
    
    ```json
    {
      "title": "Concierto de Rock",
      "date": "2026-04-15T20:00:00Z",
      "venue": "Auditorio Central",
      "description": "Gran concierto de rock con bandas locales",
      "imageUrl": "https://example.com/image.jpg",
      "active": true,
      "zones": [
        {
          "name": "Platea",
          "price": 50.00,
          "available": 100
        },
        {
          "name": "Balcón",
          "price": 30.00,
          "available": 50
        }
      ]
    }
    ```
  </Step>
  
  <Step title="Ver el Evento en la Aplicación">
    Recarga la aplicación y deberías ver tu evento en la lista principal.
    
    <Tip>
      Los eventos se cargan desde Redux y se mantienen en el estado global para acceso rápido.
    </Tip>
  </Step>
</Steps>

## Paso 6: Realizar Tu Primera Venta

<Steps>
  <Step title="Seleccionar Evento">
    En la pantalla principal, haz clic en el evento que acabas de crear.
  </Step>
  
  <Step title="Seleccionar Asientos o Zona">
    - **Con Seats.io**: Verás un mapa interactivo donde puedes seleccionar asientos específicos
    - **Sin Seats.io**: Selecciona la zona y la cantidad de entradas
    
    ```javascript
    // Ejemplo de selección con Seats.io
    import { hold_seats_seatsio } from '@/guards/firebase/firebase';
    
    const holdSeats = async (seats) => {
      const result = await hold_seats_seatsio({
        eventKey: eventId,
        seats: seats,
        holdToken: sessionToken
      });
    };
    ```
  </Step>
  
  <Step title="Ingresar Datos del Cliente">
    Completa el formulario con:
    - Nombre completo
    - Email
    - Teléfono
    - Tipo de cliente (Regular o Corporativo)
  </Step>
  
  <Step title="Seleccionar Método de Pago">
    Elige entre:
    - **Pago móvil** (con IGTF del 3%)
    - **Tarjeta de crédito/débito** (con IGTF del 3%)
    - **Efectivo** (sin IGTF)
    - **Transferencia** (sin IGTF)
    
    <Info>
      El IVA del 16% se calcula automáticamente en todos los casos.
    </Info>
  </Step>
  
  <Step title="Confirmar y Procesar">
    Revisa el resumen de la orden:
    - Entradas seleccionadas
    - Subtotal
    - IVA (16%)
    - IGTF (3% si aplica)
    - Total a pagar
    
    Haz clic en **Confirmar Venta** para procesar la transacción.
    
    ```javascript
    // La orden se procesa con Firebase Functions
    import { order_created } from '@/guards/firebase/firebase';
    
    const processOrder = async (orderData) => {
      const result = await order_created(orderData);
      // Envía email de confirmación automáticamente
    };
    ```
  </Step>
  
  <Step title="Confirmación y Email">
    Una vez procesada la venta:
    - Se genera un código QR único para cada entrada
    - Se envía un email de confirmación al cliente
    - Las entradas se marcan como vendidas en Seats.io (si aplica)
    - Se actualiza el inventario disponible
    
    <Tip>
      El cliente recibirá un email con sus entradas en formato PDF y códigos QR para validación en el evento.
    </Tip>
  </Step>
</Steps>

## Solución de Problemas Comunes

<AccordionGroup>
  <Accordion title="Error de autenticación de Firebase">
    Verifica que:
    - Las credenciales en `firebase.js` sean correctas
    - Tu proyecto de Firebase tenga Authentication habilitado
    - El método de autenticación por email/contraseña esté activado
  </Accordion>
  
  <Accordion title="No se cargan los eventos">
    Asegúrate de que:
    - La colección `events` existe en Firestore
    - Los documentos tienen el campo `active: true`
    - Las reglas de seguridad de Firestore permiten la lectura
  </Accordion>
  
  <Accordion title="Seats.io no funciona">
    Verifica:
    - Que tengas una cuenta activa de Seats.io
    - Las credenciales estén configuradas correctamente
    - El evento tenga un `eventKey` válido de Seats.io
  </Accordion>
  
  <Accordion title="Error de encriptación en Redux">
    Si ves errores de encriptación:
    - Limpia el localStorage: `localStorage.clear()`
    - Verifica que la `secretKey` esté configurada en Store.js
    - Recarga la aplicación
  </Accordion>
</AccordionGroup>
