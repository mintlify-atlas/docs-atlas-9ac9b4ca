## Introducción

El sistema TMT incluye un mecanismo de precios corporativos que permite ofrecer tarifas especiales a clientes empresariales. Este sistema se aplica automáticamente cuando se identifica una compra como corporativa.

<Info>
  Los precios corporativos se configuran a nivel de zona/categoría de asiento y son opcionales. Si no hay precio corporativo configurado, se usa el precio regular.
</Info>

## Activación del Modo Corporativo

El modo corporativo se activa mediante un selector en la interfaz de checkout:

```javascript CheckOut.js (línea 53)
const [clientType, setClientType] = useState('normal');
```

## Cálculo del Subtotal con Precios Corporativos

El cálculo del subtotal determina automáticamente qué precio usar:

```javascript CheckOut.js (líneas 712-718)
const calculateSubtotal = () => {
  const subtotal = selectedSeats.reduce((sum, seat) => {
    const price = clientType === 'corporativo' ? (seat.corporate_price || seat.price) : seat.price;
    return sum + (price || 0);
  }, 0);
  return truncateDecimals(subtotal, 2);
};
```

### Lógica de Selección de Precio

```javascript
// Pseudocódigo explicativo
if (clientType === 'corporativo') {
  if (seat.corporate_price existe y es válido) {
    usar seat.corporate_price
  } else {
    usar seat.price (fallback a precio regular)
  }
} else {
  usar seat.price
}
```

<Warning>
  Si no existe un precio corporativo configurado para una zona, se utilizará automáticamente el precio regular, sin generar errores.
</Warning>

## Aplicación en Tickets Finales

Al crear la orden, el precio corporativo se aplica individualmente a cada ticket:

```javascript CheckOut.js (líneas 332-373)
let tickets = selectedSeats.map((seat) => {
  const baseMetadata = {
    customer_name: values.customer_name,
    customer_email: values.customer_email,
    customer_phone: values.customer_phone,
    customer_id: values.customer_id,
    customer_id_type: values.customer_id_type,
    customer_address: values.customer_address,
    customer_country: countryObject,
    ...(clientType === 'corporativo' && {
      company_name: values.company_name,
      company_rif: `${values.company_rif_type}-${values.company_rif}`,
      company_address: values.company_address,
      is_corporate: true
    })
  };

  // Agregar metadata de Seats.io si aplica
  if (seat.metadata === 'Seats.io' && seat.seatsio_object) {
    baseMetadata.seatsio = {
      object_id: seat.seatsio_object.id,
      object_label: seat.seatsio_object.label,
      object_type: seat.seatsio_object.objectType,
      object_uuid: seat.seatsio_uuid,
      category_key: seat.seatsio_category,
      category_label: seat.location
    };
  }

  // La lógica corporativa se aplica aquí, no en la selección
  const finalAmount = clientType === 'corporativo' && seat.ticket_data?.corporate_price
    ? seat.ticket_data.corporate_price
    : seat.price;

  return {
    amount: finalAmount,
    id: seat.id,
    ticket_id: seat.ticket_id,
    zone: seat.location,
    metadata: baseMetadata
  }
})
```

<Note>
  Cada ticket individual guarda el monto final aplicado (corporativo o regular) en el campo `amount`.
</Note>

## Campos Requeridos para Compras Corporativas

Cuando se selecciona el tipo de cliente "corporativo", se activan campos adicionales obligatorios:

### Validación con Yup

```javascript CheckOut.js (líneas 221-230)
...(clientType === 'corporativo' && {
  company_name: Yup.string()
    .required("Se requiere el Nombre de la Empresa"),
  company_rif: Yup.string()
    .required("Se requiere el RIF de la Empresa"),
  company_rif_type: Yup.string()
    .required("Se requiere el tipo de RIF"),
  company_address: Yup.string()
    .required("Se requiere la Dirección de la Empresa")
})
```

### Campos del Formulario

<ParamField path="company_name" type="string" required>
  Nombre o razón social de la empresa
</ParamField>

<ParamField path="company_rif" type="string" required>
  Número de RIF de la empresa (sin tipo)
</ParamField>

<ParamField path="company_rif_type" type="string" required>
  Tipo de RIF: J (Jurídico), G (Gobierno), V (Venezolano), E (Extranjero)
</ParamField>

<ParamField path="company_address" type="string" required>
  Dirección fiscal de la empresa
</ParamField>

## Validación de Compras Corporativas

El sistema valida que todos los campos corporativos estén completos:

```javascript CheckOut.js (líneas 786-798)
const basicDataValid = selectedSeats.length > 0 &&
  selectedPaymentMethods.length > 0 &&
  formik.values.customer_name.trim() !== '' &&
  formik.values.customer_email.trim() !== '' &&
  formik.values.customer_phone.trim() !== '' &&
  formik.values.customer_id.trim() !== '' &&
  formik.values.customer_id_type.trim() !== '' &&
  (clientType !== 'corporativo' || (
    formik.values.company_name.trim() !== '' &&
    formik.values.company_rif.trim() !== '' &&
    formik.values.company_rif_type.trim() !== '' &&
    formik.values.company_address.trim() !== ''
  )) &&
  Math.abs(enteredAmount - finalTotalAmount) < 0.01;
```

<Warning>
  El formulario no se puede enviar hasta que todos los campos corporativos estén completos y válidos.
</Warning>

## Ejemplos de Uso

### Ejemplo 1: Compra Normal vs Corporativa

```javascript
// Configuración de zona:
const zone = {
  name: "VIP",
  price: 100.00,
  corporate_price: 85.00
};

// Compra Normal:
const normalSubtotal = 3 * zone.price;
// normalSubtotal = 3 * 100.00 = $300.00

// Compra Corporativa:
const corporateSubtotal = 3 * zone.corporate_price;
// corporateSubtotal = 3 * 85.00 = $255.00

// Ahorro:
const savings = normalSubtotal - corporateSubtotal;
// savings = $45.00 (15% de descuento)
```

### Ejemplo 2: Zona sin Precio Corporativo

```javascript
// Configuración de zona:
const zone = {
  name: "General",
  price: 50.00,
  corporate_price: null  // No hay precio corporativo
};

// Compra Corporativa (usa precio regular):
const corporateSubtotal = 2 * (zone.corporate_price || zone.price);
// corporateSubtotal = 2 * 50.00 = $100.00
// Usa precio regular porque corporate_price es null
```

### Ejemplo 3: Compra Mixta de Zonas

```javascript
// Cliente corporativo compra asientos de diferentes zonas:

const selectedSeats = [
  { zone: "VIP", price: 100.00, corporate_price: 85.00 },
  { zone: "VIP", price: 100.00, corporate_price: 85.00 },
  { zone: "Platea", price: 60.00, corporate_price: null },
];

// Cálculo:
const subtotal = 
  85.00 +  // VIP corporativo
  85.00 +  // VIP corporativo
  60.00;   // Platea (sin precio corporativo, usa regular)

// subtotal = $230.00
```

## Reportes y Análisis

El campo `is_corporate` en las órdenes permite:

1. **Generar reportes** de ventas corporativas vs. individuales
2. **Analizar descuentos** otorgados a clientes corporativos
3. **Segmentar clientes** para campañas de marketing
4. **Calcular comisiones** diferenciadas
5. **Exportar datos fiscales** para facturación

```javascript
// Ejemplo de query para reportes:
const corporateOrders = orders.filter(order => order.is_corporate === true);
const totalCorporateSales = corporateOrders.reduce((sum, order) => sum + order.amount, 0);
```

## Consideraciones Fiscales

<Warning>
  Las compras corporativas en Venezuela tienen implicaciones fiscales especiales:
  
  - Requieren factura con RIF
  - Pueden ser deducibles de impuestos para la empresa
  - Deben cumplir con normativas del SENIAT
  - El IVA es recuperable para empresas registradas
</Warning>
