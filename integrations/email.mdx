## Descripción General

TMT - Taquilla Física incluye un sistema completo de notificaciones por correo electrónico que envía confirmaciones de compra detalladas a los clientes después de completar una transacción.

<Note>
Los correos electrónicos se envían automáticamente mediante Firebase Cloud Functions utilizando un servicio de correo transaccional.
</Note>

## Características

El sistema de correos incluye:

- Confirmaciones de compra con diseño profesional
- Detalles completos del evento y asientos
- Información del comprador (individual o corporativo)
- Resumen de transacciones y métodos de pago
- Cálculo de impuestos (IVA, IGTF)
- Enlaces de descarga de la app móvil
- Plantillas HTML responsive

## Configuración

### Cloud Function de Envío

La función `send_email` está declarada en Firebase:

```javascript firebase.js
import { getFunctions, httpsCallable } from "firebase/functions";

const functions = getFunctions(app);
const send_email = httpsCallable(functions, "send_email");

export { send_email };
```

### Parámetros de la Función

<CodeGroup>
```javascript Estructura de Parámetros
send_email({
  to_email: string,      // Email del destinatario
  to_name: string,       // Nombre del destinatario
  html_send: string,     // Contenido HTML del email
  subject: string        // Asunto del email
})
```
</CodeGroup>

## Implementación

### Envío de Confirmación de Compra

Después de crear una orden exitosamente, se envía el correo de confirmación:

```javascript CheckOut.js
import { send_email } from '../../guards/firebase/firebase';
import moment from 'moment';
import 'moment/locale/es';

// Después de crear la orden
const orderId = resp.data.order || 'ORD-' + Date.now().toString().substring(5);
const customerEmail = values.customer_email;
const customerName = values.customer_name;
const subject = `Confirmación de compra: ${eventData?.event?.name || 'Evento'}`;

// Formatear fecha del evento
const eventDate = eventData?.date_start 
  ? moment(eventData.date_start.toDate ? eventData.date_start.toDate() : eventData.date_start)
      .locale('es')
      .format('DD [de] MMMM [de] YYYY') 
  : 'Fecha no disponible';

const eventTime = eventData?.date_start 
  ? moment(eventData.date_start.toDate ? eventData.date_start.toDate() : eventData.date_start)
      .locale('es')
      .format('h:mm a') 
  : '';

// Construir HTML del email
const htmlContent = `...`; // Ver plantilla completa abajo

// Enviar email
send_email({
  to_email: customerEmail,
  to_name: customerName,
  html_send: htmlContent,
  subject: subject
}).then(() => {
  toast.info('Correo de confirmación enviado.');
}).catch((error) => {
  console.error('Error al enviar el correo para la orden:', orderId, error);
  toast.error('No se pudo enviar el correo de confirmación.');
});
```

## Cálculos de Impuestos

El resumen incluye desglose de impuestos:

- **Subtotal**: Suma de precios de tickets
- **IVA (16%)**: Impuesto al Valor Agregado
- **IGTF (3%)**: Impuesto a Grandes Transacciones Financieras (condicional)
- **Total en USD**: Total final en dólares
- **Total en BS**: Total convertido a bolívares


## Solución de Problemas

### Email no se envía

<Steps>
  <Step title="Verificar Cloud Function">
    Confirma que la función `send_email` esté desplegada
  </Step>
  
  <Step title="Revisar Logs">
    Consulta los logs de Firebase Functions
  </Step>
  
  <Step title="Verificar Cuota">
    Asegúrate de no haber excedido límites de envío
  </Step>
</Steps>

### Email llega a spam

Para evitar que los emails lleguen a spam:

1. Configura SPF y DKIM en tu dominio
2. Utiliza un servicio de email transaccional reputado
3. Evita palabras spam en el asunto
4. Mantén una ratio baja de rebotes

### Formato incorrecto

Si el email no se ve bien:

1. Verifica que uses estilos inline
2. Evita CSS complejo
3. Prueba en múltiples clientes de email
4. Usa herramientas como Litmus para testing
