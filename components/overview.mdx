## Introducción

TMT utiliza una biblioteca de componentes UI construidos sobre [Radix UI](https://www.radix-ui.com/) y estilizados con [Tailwind CSS](https://tailwindcss.com/). Los componentes están diseñados para ser accesibles, personalizables y fáciles de usar.

## Arquitectura

Todos los componentes UI siguen un patrón consistente:

```
src/components/
├── ui/           # Componentes base reutilizables
│   ├── Button.js
│   ├── Input.js
│   ├── Card.js
│   └── ...
└── apps/        # Componentes específicos de la aplicación
    ├── events/
    ├── auth/
    └── ...
```

## Utilidad cn()

Los componentes usan la utilidad `cn()` para combinar clases de Tailwind:

```javascript src/utils/cn.js
import { clsx } from "clsx";
import { twMerge } from "tailwind-merge";

export function cn(...inputs) {
  return twMerge(clsx(inputs));
}
```

Esto permite combinar clases condicionales sin conflictos.
